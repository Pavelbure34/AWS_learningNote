day04.Actual Database Practice

    What we will do here
  1. we will create SQL database in VPC
  2. And get access to it remotely
  3. Do all the operations!

1.Create SQL db in VPC

  1. If you haven’t created one,create it.
  2. Select database type.
    • Since we only practice it, find free tier.
  3. For now,check Instance class. Choose db.t2.micro.
  4. Name of the database has to be unique.
  5. Don’t bother changing advanced settings for now!

2.remote access to SQL db in VPC

  1. Now db created,check for end point.
    • it is going to be the access point to this.
  2. Open MySQL workBench for access to your DB.
  3. For creating SQL server, use end point of your RDS as host name.
  4. For xmaster user for local MYSQL, use the same name of master user for RD and password for remote connection.

    Key terms

  • AWS cloud =a global infrastructure that provides all the essential IT resources and services to any developers.
  • Virtual Private Cloud(VPC)= privately owned and protected space within AWS cloud.
      Properties
    • Can only access by you or anyother allowed individuals.
  • Cloud Computing Models
    • Infrastructure as a Service (IaaS)
    • Platform as a Service( PaaS)
      • It has more control than IaaS
    • Software as a Service( SaaS)
      • It has complete control over services.
    • Serverless Computing or Function as a Service ( FaaS)
        Properties
      • Does not have to worry about server settings.
      • Examples
        • AWS lambda: able to run code within the server.
